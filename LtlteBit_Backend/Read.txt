## 📝 Summary: Node, npm, TypeScript, and Configs

### 1. Node.js
- **What**: A JavaScript runtime that executes `.js` files.  
- **Why**: It doesn’t need extra metadata — just runs the code.  
- **How**:  
  ```bash
  node app.js
  ```
  Works fine without `package.json` if no external modules are used.

---

### 2. npm & `package.json`
- **What**: npm is the package manager; `package.json` is its manifest file.  
- **Why**: Needed to track dependencies, scripts, and project metadata.  
- **How**:  
  - If your project uses external modules (`express`, `lodash`, etc.), you must have `package.json`.  
  - Example:
    ```bash
    npm init -y
    npm install express
    ```
    → `express` gets recorded in `dependencies`.

---

### 3. `npm install --save-dev`
- **What**: Installs a package as a **development dependency**.  
- **Why**: Keeps production lean; marks tools like `jest`, `eslint`, `webpack` as dev-only.  
- **How**:  
  ```bash
  npm install jest --save-dev
  ```
  → Stored under `devDependencies` in `package.json`.

---

### 4. TypeScript & `tsconfig.json`
- **What**: TypeScript compiler (`tsc`) converts `.ts` → `.js`.  
- **Why**: `tsconfig.json` defines compiler options (target, module system, strictness, output dir).  
- **How**:  
  - Without `tsconfig.json`:  
    ```bash
    npx tsc app.ts
    ```
    Compiles with defaults (old ES3, CommonJS).  
  - With `tsconfig.json`:  
    ```bash
    npx tsc
    ```
    Compiles the whole project using config rules.  
  - If you run `npx tsc app.ts` while `tsconfig.json` exists → error (`TS5112`), because you’re mixing project mode with single-file mode.

---

### 5. Putting It Together
- **Node only** → Just `.js` files, no `package.json`.  
- **Node + npm** → Needs `package.json` to manage dependencies.  
- **Node + TypeScript** → Usually has both:
  - `package.json` (dependencies, scripts)  
  - `tsconfig.json` (compiler options)

---

👉 **Golden Rule**:  
- **No external modules → no `package.json` needed.**  
- **External modules → `package.json` required.**  
- **TypeScript projects → `tsconfig.json` recommended for consistency.**
