An **AJAX request** (short for *Asynchronous JavaScript and XML*) is a way for web pages to communicate with a server **without reloading the entire page**. It’s a technique used in modern web development to make applications feel faster and more interactive.

Here’s the breakdown:

- **Asynchronous**: The request happens in the background, so the user can keep interacting with the page while data is being fetched or sent.
- **JavaScript**: AJAX relies on JavaScript to send and handle the request.
- **XML (or JSON today)**: Originally, data was exchanged in XML format, but nowadays JSON is far more common because it’s lighter and easier to work with.

### Example Flow
1. A user clicks a button on a webpage.
2. JavaScript sends an AJAX request to the server (e.g., asking for new data).
3. The server processes the request and sends back a response (often JSON).
4. JavaScript updates part of the webpage with the new data — **without refreshing the whole page**.

### Simple Example (using `fetch` in modern JS)
```javascript
fetch('/api/data')
  .then(response => response.json())
  .then(data => {
    document.getElementById('output').innerText = data.message;
  })
  .catch(error => console.error('Error:', error));
```

This snippet sends an AJAX request to `/api/data`, gets JSON back, and updates the page dynamically.
