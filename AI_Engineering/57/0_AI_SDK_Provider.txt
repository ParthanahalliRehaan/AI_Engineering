## 🧩 What is `ai-sdk-provider`?
- The **AI SDK** is a model‑agnostic toolkit that lets you build agents, tools, and workflows.  
- **Providers** are plug‑ins (modules) that connect the SDK to actual model backends.  
- Example:  
  - `@openai/ai-sdk-provider` → connects to OpenAI models.  
  - `@anthropic/ai-sdk-provider` → connects to Anthropic’s Claude models.  
  - `@openrouter/ai-sdk-provider` → connects to **OpenRouter**, which itself gives you access to many different model providers (OpenAI, Anthropic, Mistral, etc.) through one unified API.

So `ai-sdk-provider` is the **adapter layer** that tells the SDK how to talk to a specific model host.

---

## ⚡ Why `@openrouter/ai-sdk-provider` is special
- **OpenRouter** is like a “hub” for LLMs.  
- Instead of signing up for each provider separately, you can route requests through OpenRouter.  
- With `@openrouter/ai-sdk-provider`, you can use the same SDK code and swap between models like:
  - `openrouter("openai/gpt-4o-mini")`  
  - `openrouter("anthropic/claude-3-opus")`  
  - `openrouter("mistral/mistral-large")`  

All without changing your agent logic.

---

## 🛠️ Example: Using `@openrouter/ai-sdk-provider`

```js
import { generateText } from "ai";
import { createOpenRouter } from "@openrouter/ai-sdk-provider";

// Create provider instance
const openRouter = createOpenRouter({
  apiKey: process.env.OPENROUTER_API_KEY
});

// Define a tool
const weatherTool = {
  name: "weather",
  description: "Get the weather for a location",
  parameters: {
    type: "object",
    properties: { location: { type: "string" } },
    required: ["location"]
  },
  execute: async ({ location }) => {
    return { location, temperature: 72 };
  }
};

// Run agent with OpenRouter provider
async function runAgent() {
  const result = await generateText({
    model: openRouter("openai/gpt-4o-mini"), // routed via OpenRouter
    tools: [weatherTool],
    prompt: "What's the weather in New York?",
    stopWhen: 2
  });

  console.log("Final Answer:", result.text);
  console.log("Tool Calls:", result.toolCalls);
  console.log("Tool Results:", result.toolResults);
}

runAgent();
```

## 🎯 Why this matters in context engineering
- **Portability**: You design your agent once, then swap providers easily.  
- **Flexibility**: You can experiment with different models (OpenAI, Anthropic, Mistral, etc.) without rewriting your workflow.  
- **Context engineering**: You focus on tools, schemas, and retrieval logic — the provider handles the connection to the model backend.  

---

## ✅ Takeaway
- `ai-sdk-provider` modules are **adapters** for the AI SDK.  
- `@openrouter/ai-sdk-provider` is especially useful because it gives you access to **multiple model providers through OpenRouter**.  
- This means you can build one agent and run it on OpenAI today, Anthropic tomorrow, or Mistral next week — all with the same code.
