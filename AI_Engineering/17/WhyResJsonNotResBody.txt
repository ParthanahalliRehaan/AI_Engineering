🌍 In Express (Node.js)
- You often see:
  app.post("/endpoint", (req, res) => {
    console.log(req.body);
  });
- Here, req.body is populated by middleware (like express.json()).
- Express automatically reads the raw stream, parses JSON, and attaches the result to req.body.
- So in traditional server frameworks, req.body is already a parsed object because the framework did the work for you.

☁️ In Cloudflare Workers (Fetch API)
- You don’t have Express middleware.
- The Request object is a web standard, not an Express wrapper.
- request.body is just a ReadableStream of raw bytes.
- That’s why you need to call:
  const data = await request.json();
- This consumes the stream and parses JSON for you.
- If you tried to use request.body directly, you’d just get a stream, not a usable object.

✅ Summary
- Express/Node.js: req.body works because middleware parses it for you.
- Cloudflare Workers / Fetch API: you must use request.json() (or request.text() + JSON.parse) because there’s no middleware — you’re working directly with the raw web standard.

💡 Analogy
- Express = “helper waiter” who brings your meal already plated (req.body).
- Cloudflare Worker = “raw kitchen” where you have to cook the ingredients yourself (request.json() does the cooking for you).
