🔑 POST vs GET in client-server communication

- POST is used when:
  • You need to send data from the client to the server (e.g., user input, form data).
  • The server processes that data and returns a result.
  • The request body carries JSON or other payloads.
  • Typical for actions like creating, updating, or triggering AI responses.

- GET is used when:
  • You only want to retrieve data or resources without sending a body.
  • Parameters are passed in the URL query string.
  • Typical for fetching static files, public APIs, or simple lookups.

⚖️ Why POST in your case:
- The backend route is defined as `app.post("/api/gift-genie")`.
- It expects `req.body.prompt`, which only works with POST (since GET has no body).
- You’re sending user input (the gift prompt) → POST is correct.

📌 Rule of thumb:
- If the server expects `req.body` → use POST.
- If the server expects query params (`req.query`) → use GET.
