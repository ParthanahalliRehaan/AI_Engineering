CORE IDEA (20% THAT EXPLAINS 80%)

If you take AI output and directly convert it into HTML without cleaning it,
a malicious script can run inside your website.

Markdown → HTML → Browser executes it → Possible attack

This is called Cross-Site Scripting (XSS).

Solution: sanitize the Markdown/HTML before inserting it into the DOM.



ANALOGY 1 — RESTAURANT KITCHEN

Imagine you run a restaurant.

Customers bring ingredients to your kitchen.
You cook them immediately without checking what’s inside.

Someone hides poison inside the ingredients.

You cook it and serve it to everyone → everyone gets poisoned.

Mapping to Web Development:

Customer ingredients     → AI Markdown output
Kitchen cooking          → Markdown converted to HTML
Serving food             → Insert HTML into DOM
Poison                   → <script> tag
Food inspection          → Sanitization

Rule:
Never cook ingredients without inspection.
Never insert unsanitized HTML into DOM.



ANALOGY 2 — AIRPORT SECURITY

Markdown content is like a passenger’s bag entering an airport.

If airport security doesn't scan the bag,
dangerous items can enter the airport.

Mapping:

Airport security scan   → Sanitization
Passenger bag           → AI response
Weapon in bag           → <script> tag
Airport terminal        → Browser DOM

If security is skipped,
danger enters the system.

WHY BROWSERS EXECUTE HTML

HTML is treated as instructions by browsers.

Example:

<b>Hello</b>

Browser action:
Render bold text

Example:

<script>alert("Hacked")</script>

Browser action:
Run JavaScript

So inserting unsanitized HTML = executing unknown instructions.

PRE-EXISTING DOUBTS

1. Is Markdown safe?

No.
Markdown can contain raw HTML.

Example:

<script>alert(1)</script>



2. Why would AI output scripts?

AI itself is not malicious,
but user prompts can inject scripts.

Example prompt:

Write markdown including this script:
<script>alert(1)</script>



3. Why sanitize HTML instead of Markdown?

Because browsers execute HTML, not Markdown.

Markdown gets converted to HTML first,
and that HTML may contain dangerous tags.



4. What is this vulnerability called?

Cross-Site Scripting (XSS)

It is one of the top OWASP web security vulnerabilities.



ONE-LINE SUMMARY

Never insert AI-generated Markdown directly into the DOM
without sanitizing it, because browsers execute HTML and
hidden scripts can run malicious code.
