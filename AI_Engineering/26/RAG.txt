### 1. What is “context” for an LLM?

When an LLM answers, it only sees the text you send **right now** in the prompt.  
That visible text (system prompt + your question + any extra info you add) is called the **context**. [towardsdatascience](https://towardsdatascience.com/understanding-context-and-contextual-retrieval-in-rag/)

Example:  
If you send:

> “You are a helpful tutor.  
> Here is a document about my company…  
> User: What is our refund policy?”

All of that text is the **context** the model uses to generate the next tokens. [cloud.google](https://cloud.google.com/use-cases/retrieval-augmented-generation)

### 2. What is RAG?

**RAG = Retrieval-Augmented Generation**.  

- **Retrieval**: Before answering, the system **retrieves** relevant pieces of data (e.g., from a vector database) that match your question. [aws.amazon](https://aws.amazon.com/what-is/retrieval-augmented-generation/)
- **Augmented Generation**: It then **adds** those pieces into the prompt as extra context and only then asks the LLM to generate an answer. [en.wikipedia](https://en.wikipedia.org/wiki/Retrieval-augmented_generation)

So RAG is basically:  
1. Find relevant info from your knowledge base.  
2. Attach it to the prompt as context.  
3. Let the LLM answer using that context.

### 3. Your sentence, unpacked

> “Those chunks are then fed into the LLM as context (RAG), so the answer is grounded in your data, not random internet knowledge.”

- “Those chunks” = the text pieces retrieved from the vector DB.  
- “Fed into the LLM as context” = they are added to the prompt, so the model can read them. [alanknox](https://alanknox.com/adding-context-to-the-context-in-retrieval-augmented-generation-rag/)
- “(RAG)” = this whole process of retrieving + adding to the prompt. [aws.amazon](https://aws.amazon.com/what-is/retrieval-augmented-generation/)
- “Grounded in your data” = the model answers using **your documents**, instead of just guessing from its training. [blogs.nvidia](https://blogs.nvidia.com/blog/what-is-retrieval-augmented-generation/)

So, in simple terms:

> RAG means: “Before the model answers, first look up the most relevant docs, put them in the prompt, then answer using them.”
