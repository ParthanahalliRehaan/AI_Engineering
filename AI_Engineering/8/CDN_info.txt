================================
CDN NOTES (CONTENT DELIVERY NETWORK)
================================

1. FULL FORM
--------------------------------
CDN = Content Delivery Network


2. WHAT IS A CDN
--------------------------------
A CDN is a distributed network of servers that stores copies of files
(JavaScript, CSS, images, fonts, videos) and delivers them to users
from the server closest to their geographic location.

Instead of loading files from one central server,
the browser downloads them from the nearest available CDN server.


3. WHY CDN EXISTS
--------------------------------
Problem without CDN:

User (India) → Request → Server (USA)

Issues:
- High latency
- Slow loading
- Heavy load on one server


CDN Solution:

User (India) → India CDN server
User (Europe) → Europe CDN server
User (USA) → USA CDN server

Result:
- Faster response
- Reduced server load
- Better reliability


4. WHAT KIND OF FILES CDN HOSTS
--------------------------------

Common CDN content:

• JavaScript libraries
• CSS frameworks
• Images
• Videos
• Fonts

Examples:

React
Vue
Bootstrap
Marked
jQuery


5. EXAMPLE OF CDN LINK
--------------------------------

https://cdn.jsdelivr.net/npm/marked/marked.min.js


6. HOW CDN IS USED IN HTML
--------------------------------

Example:

<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>

When the browser loads the page:

Step 1
Browser requests file from CDN

Step 2
Nearest CDN server responds

Step 3
File is downloaded and executed

Step 4
Library functions become available in browser


7. GLOBAL VARIABLE CONCEPT
--------------------------------

Most CDN libraries expose a global variable.

Example with marked:

<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>

Now the browser has access to:

marked.parse("# Hello")


8. IMPORTANT RULES FOR CDN
--------------------------------

Rule 1
One <script> tag must have only ONE src.

Rule 2
CDN script must load BEFORE your script.

Rule 3
Do not use npm import with CDN libraries.

Rule 4
CDN libraries create global variables.


9. CDN VS LOCAL FILE VS BUNDLER
--------------------------------

CDN

<script src="cdn-url"></script>

Library is loaded from the internet.


LOCAL FILE

<script src="/js/marked.js"></script>

Library is stored inside your project.


BUNDLER

import { marked } from "marked"

Bundler compiles dependencies for the browser.


10. ADVANTAGES OF CDN
--------------------------------

• Faster loading using nearby servers
• No installation required
• Easy to use
• Reduces load on origin server
• Browser caching


11. DISADVANTAGES OF CDN
--------------------------------

• Requires internet connection
• Version changes may break code
• External dependency
• Less control over library versions
• Security risks if CDN is compromised

13. REAL LIFE ANALOGY
--------------------------------

CDN works like multiple warehouses.

Without CDN:

All products ship from one warehouse.

With CDN:

Warehouses exist in different countries.
Customers receive products from the nearest warehouse.

CDN servers act like these warehouses for files.


14. FLOW OF CDN
--------------------------------

Browser
   │
   ▼
Request CDN file
   │
   ▼
Nearest CDN server responds
   │
   ▼
File downloaded
   │
   ▼
Browser executes library
   │
   ▼
Functions become available globally


15. THE 20% THAT EXPLAINS 80%
--------------------------------

Understand these core ideas:

1. CDN hosts libraries online.
2. Browsers download files directly from CDN servers.
3. CDN libraries expose global variables.
4. Script loading order matters.
5. CDN avoids installing libraries locally.
