SUMMARY: FASTAPI vs NODE.JS — ENDPOINTS & DEPLOYMENT

1. CORE RULE
   - Browser runs JavaScript
   - Server runs Python or Node.js
   - Communication happens only via HTTP endpoints

   Endpoint structure:
   protocol + host + port + path

2. WHAT CHANGES AFTER DEPLOYMENT
   - Only HOST (and sometimes PORT) changes
   - PATH never changes
   - Backend code stays the same
   - Frontend fetch URL changes

   Example:
   Local:     http://localhost:8000/users
   Deployed:  https://api.onrender.com/users

3. WHY FASTAPI FEELS "SEPARATE"
   - FastAPI is API-first
   - Usually does NOT serve HTML
   - Frontend and backend are on different origins
   - Frontend must use absolute URLs
   - CORS is required
   - This is standard industry architecture

4. WHY NODE.JS FEELS "UNIFIED"
   - Express often serves both:
       "/"       → HTML
       "/api/*"  → API
   - Same server, same origin
   - Frontend uses relative paths:
       fetch("/api/users")
   - No endpoint change after deployment
   - This is a monolithic setup

5. NODE.JS WHEN SEPARATED (IMPORTANT)
   - If Node backend and frontend are on different servers:
       fetch("https://api.myapp.com/users")
   - Same behavior as FastAPI
   - Absolute URLs required
   - CORS required

6. THREE COMMON ARCHITECTURES

   A) MONOLITH (Node typical)
      myapp.com
       ├── /        → frontend
       └── /api     → backend
      - Relative URLs
      - No CORS
      - No endpoint changes

   B) SEPARATED (FastAPI typical)
      app.myapp.com  → frontend
      api.myapp.com  → backend
      - Absolute URLs
      - Env variables
      - CORS required

   C) PRO SETUP (Reverse Proxy)
      myapp.com
       ├── /        → frontend
       └── /api     → backend
      - Works with Node or FastAPI
      - No CORS
      - Industry standard

7. BEST PRACTICE
   - Never hardcode URLs
   - Use environment variables
   - Change config, not code

8. FINAL TRUTH
   - Node.js hides architecture early
   - FastAPI exposes real-world architecture early
   - Difference is architecture, not language
