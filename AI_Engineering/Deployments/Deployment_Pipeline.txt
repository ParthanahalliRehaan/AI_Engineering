## 🔄 What a Deployment Pipeline Is
A deployment pipeline is the automated sequence of steps that takes your code from GitHub (or another repo) and turns it into a live site. Think of it as a factory line:

1. **Source retrieval** → Pulls your latest code from GitHub.
2. **Install dependencies** → Runs `npm install` (or equivalent) to get all libraries.
3. **Build step** → Executes your build command (`npm run build` for Vite).
4. **Collect artifacts** → Grabs the output folder (`dist/`).
5. **Deploy to CDN** → Pushes those files to Cloudflare’s global edge network.
6. **Serve traffic** → Your site is live at `project.pages.dev` (or custom domain).

---

## ⚙️ Cloudflare Pages Pipeline (with Vite)
Here’s how it looks specifically for your case:

- **Trigger**: You push to GitHub → Cloudflare notices.
- **Step 1: Clone repo** → Cloudflare pulls the `main` branch.
- **Step 2: Install** → Runs `npm install` to fetch Vite + dependencies.
- **Step 3: Build** → Runs `npm run build`.  
  - Vite generates the `dist/` folder.
- **Step 4: Deploy** → Cloudflare Pages takes everything inside `dist/` and distributes it across their edge servers.
- **Step 5: Serve** → Requests to your site are answered from the nearest Cloudflare location.

---

## 🆚 Render vs. Cloudflare
- **Render (static hosting)**: If you already have plain HTML/CSS/JS, Render just copies those files to its servers. No build pipeline needed.
- **Cloudflare Pages (with Vite)**: Since you’re using a bundler, Cloudflare must run the build pipeline to transform your source into deployable assets.

---

## 🎯 Why Pipelines Matter
- **Automation**: No manual “build → upload” steps.
- **Consistency**: Every push goes through the same process, reducing errors.
- **Continuous Deployment (CD)**: Your site updates automatically with each commit.
- **Security**: API keys and environment variables are injected safely during build.

---

👉 So in short: the pipeline is Cloudflare’s way of taking your *source code* and turning it into *production-ready files* every time you push. That’s why `npm run build` and `dist/` are central — they’re the “assembly line” outputs that Cloudflare needs to serve your app.
