Good question—what you call it depends on how you’re running the model:

- **Earlier (API approach):** You were using the **Hugging Face Inference API**. That’s a *remote* service where Hugging Face hosts the model and you send requests over HTTP.  
- **Now (local approach):** You’re using **`transformers.js`**, which is Hugging Face’s JavaScript library for running models directly on your machine (Node.js or browser). This is often referred to as **local inference** or **on‑device inference**.

So if you want to describe the shift, you can say something like:

- *“I was using Hugging Face’s hosted Inference API, but now I’m running models locally with `transformers.js` for on‑device feature extraction.”*  
- Or simply: *“I moved from remote inference to local inference.”*

---

👉 Since you’re experimenting with embeddings, this distinction matters:  
- **Remote inference** → Hugging Face does the heavy lifting, you just consume results.  
- **Local inference** → You control the runtime, caching, and performance trade‑offs, and you don’t need an API token.
