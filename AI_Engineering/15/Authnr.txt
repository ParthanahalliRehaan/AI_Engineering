---

### 🔑 Authentication vs Authorization
- **Authentication** = *“Who are you?”*  
  It’s the process of verifying identity (e.g., logging in with a username/password, fingerprint, or token).
- **Authorization** = *“What are you allowed to do?”*  
  Once identity is confirmed, authorization decides what resources or actions that identity can access.

---

### 🖥️ Local Authentication
This means the identity check happens **on your own machine or server**, not through a cloud service like Supabase or Firebase.

Examples:
- **Local database of users**: You store usernames and hashed passwords in a local Postgres/MySQL/SQLite DB.
- **File-based auth**: A simple JSON or config file with user credentials.
- **OS-level auth**: Using system accounts (like Linux PAM or Windows users).
- **Local token generation**: Your app issues JWTs (JSON Web Tokens) after login, without calling an external API.

So instead of Supabase handling login, *your code* checks the credentials against a local source.

---

### 🛡️ Local Authorization
This is about enforcing rules locally once a user is authenticated.

Examples:
- **Role-based access control (RBAC)**: Users have roles (admin, editor, viewer) stored in your local DB. Your app checks the role before allowing actions.
- **Permission tables**: A local table defines which user IDs can access which resources.
- **Hardcoded rules**: In small apps, you might just write logic like `if user.isAdmin then allow`.

---

### ⚖️ Why it matters
- **Local = full control**: You decide how credentials are stored, how tokens are issued, and how permissions are enforced.
- **Cloud = convenience**: Supabase/Firebase handle all of this for you, securely and at scale.
- **Hybrid approach**: Many apps start with cloud auth (fast, secure) and later add local auth for offline use, custom rules, or advanced control.

---

Think of it like this:  
- **Authentication** is the lock on the door.  
- **Authorization** is deciding which rooms inside the house you can enter.  
- Doing it **locally** means you build and manage the locks and keys yourself, instead of renting them from Supabase.
