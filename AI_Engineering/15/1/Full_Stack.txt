Core concept (locked-in)
Frontend is a 2D side‑scroller: move → learn concept → move → mini‑boss → final boss; each boss requires answering questions + writing code.

Coding happens in an HTML/JS editor panel, not inside the canvas: easier to build, easier to store/validate, more “real dev” feel.

Every 24 hours the user gets one “day quest” split into 4 bosses, tied to concepts your AI (or rules) pick.

Users:

Create account (auth).

Play game; each cleared boss is stored in DB with XP.

Progress, streaks, and unlocked levels are all persisted.

Frontend & game stack
Framework:

Next.js (App Router) + TypeScript – one app for pages + API routes + game page.

Styling & UI:

Tailwind CSS for layouts + responsive UI.

shadcn/ui (or similar) for modals, buttons, forms, to avoid custom UI boilerplate.

2D game engine (recommended): Phaser 3

Use the official Phaser 3 + Next.js template so React and Phaser are already wired together properly.

Game runs in a <canvas> inside a page like /play, controlled via a Phaser Scene.

Alternative (simpler engine): Kaboom.js

Great for simple platformers and used in dev‑portfolio‑as‑game examples.

Still integrates via a <canvas> on a Next.js page.

Code editor UI:

Use Monaco Editor (VS Code’s editor) embedded in a React component that appears when a boss fight starts.

Flow: Phaser triggers “boss fight” → React shows modal with Monaco editor + questions; on success, call API, then notify Phaser that boss is defeated.

Backend & API design
Runtime:

Next.js Route Handlers (API routes) in TypeScript – you don’t need a separate backend server initially.

API responsibilities:

Auth‑protected endpoints for:

Creating/updating profile.

Fetching today’s 4 bosses + tasks.

Validating challenge (optional AI) and marking boss as beaten.

Updating XP, streaks, unlocked levels.

Internal API style:

REST (simple) or tRPC if you want typesafe end‑to‑end; REST is enough and interviewers know it well.

Data & game logic layer
Database:

PostgreSQL on Supabase or Neon.

ORM:

Prisma for schema, migrations, and type‑safe queries.

Tables (high level):

users – auth user id, profile info.

skills – per topic skill XP/level (html, css, js, backend, ai).

day_quests – one row per user per day (date, status).

bosses – four per day quest, with concept, difficulty, and status.

boss_attempts – code submitted, result (pass/fail), timestamps.

xp_history – how much XP earned from which boss/quest.

Game state sync:

On loading /play, frontend calls /api/today-quest to get bosses/progress and initialize Phaser state.

When player reaches a boss gate, Phaser/React uses this data to show the relevant challenge; on success → call /api/boss/complete.

AI usage (optional but impressive)
Where AI fits (if you want it):

Generate boss questions and code prompts per concept.

Provide hints/explanations if the learner fails multiple times.

Later: auto‑evaluate simple answers or code (for controlled tasks).

Stack:

Hosted LLM (OpenAI / Anthropic / etc.).

Light agent/orchestrator (LangChain/LangGraph) for a “Boss Generator” and (maybe) “Answer Evaluator” service.

You can start rule‑based (manual questions stored in DB) and bolt AI on later.

Auth & security
Auth options (pick one):

Auth.js (NextAuth) – tightly integrated with Next.js, supports email/password + OAuth.

Clerk – great DX and built‑in UIs if you want to move faster.

Security basics:

All write APIs require auth; use middleware in Next.js to protect /play and /api/*.

Store user IDs in DB FK; no trusting client for XP or progress.

Rate limit critical routes (login, boss validation) using Redis / Cloudflare KV.

Deployment & “good servers”
App hosting:

Vercel or Cloudflare Pages/Workers for Next.js + Phaser game. Both can handle hundreds of users easily.

Database:

Supabase or Neon for managed Postgres (auto scaling, backups).

Background work:

Vercel Cron or Cloudflare Cron to:

Generate daily quests (4 bosses) for each active user at midnight.

Maybe recalc XP/skill levels.

Caching & rate limiting:

Upstash Redis or Cloudflare KV for caching AI responses and rate limiting.

Monitoring:

Sentry for errors, PostHog/Plausible for analytics (how many users reach boss 4, etc.).

TL;DR stack (what you’d put on resume)
Frontend & game: Next.js (TS), Tailwind, shadcn/ui, Phaser 3 (2D game with bosses, integrated via official Phaser+Next template).

Backend: Next.js API routes (TS), REST, Auth.js/Clerk.

Data: Postgres (Supabase/Neon) + Prisma, structured around users, daily quests, bosses, XP, attempts.

AI (optional but strong signal): hosted LLM + simple agent for generating challenges/adaptive difficulty.

Infra: Vercel/Cloudflare deploy, cron jobs for daily quests, Redis/KV for caching/rate limiting, Sentry + PostHog/Plausible for observability.
