GOAL
-----
Generate structured JSON from an LLM using a JSON Schema.

------------------------------------------------------------
1. WHY YOU GOT THIS OUTPUT
------------------------------------------------------------

Output:
{
  "gifts": []
}

Reason:
Your schema allowed an empty array.

Schema rules:

{
  "gifts": {
    "type": "array"
  }
}

This means:
✔ gifts must exist
✔ gifts must be an array
✘ gifts do NOT need elements

Therefore the model chooses the safest valid response:

{ "gifts": [] }

LLMs always prefer the **minimum valid output**.


------------------------------------------------------------
2. FIX → FORCE ITEMS IN ARRAY
------------------------------------------------------------

Add:

minItems

Correct schema:

{
  "type": "json_schema",
  "name": "gift_suggestions",
  "schema": {
    "type": "object",
    "additionalProperties": false,

    "properties": {
      "gifts": {
        "type": "array",
        "minItems": 5,

        "items": {
          "type": "object",
          "additionalProperties": false,

          "properties": {
            "name": { "type": "string" },
            "price_range": { "type": "string" },
            "why_its_good": { "type": "string" }
          },

          "required": [
            "name",
            "price_range",
            "why_its_good"
          ]
        }
      }
    },

    "required": ["gifts"]
  }
}

Now this becomes INVALID:

{ "gifts": [] }

Model must generate objects.


------------------------------------------------------------
3. GOOD PROMPT PRACTICE
------------------------------------------------------------

Bad prompt:

Make an essay about Shogatsu


Good prompt:

Suggest exactly 5 gifts for Japanese New Year (Shogatsu).
Each gift must include:
- name
- price_range
- why_its_good


Schema + Prompt must match.


------------------------------------------------------------
4. CORRECT NODEJS IMPLEMENTATION
------------------------------------------------------------

const response = await openAI.responses.create({
  model: process.env.AI_MODEL,

  input: `
  Suggest exactly 5 gifts for Japanese New Year (Shogatsu).
  Include name, price_range and why_its_good.
  `,

  text: {
    format: giftSchemaResponses
  }
});

console.log(response.output_parsed);


IMPORTANT

Do NOT do this:

JSON.parse(response.output_text)

Responses API already parses the JSON.

Use:

response.output_parsed


------------------------------------------------------------
5. HOW TO READ JSON SCHEMA (MENTAL MODEL)
------------------------------------------------------------

STEP 1 — type

type: object

means:

{ }


STEP 2 — properties

properties:
  gifts

means:

{
  "gifts": ...
}


STEP 3 — array

type: array

means:

{
  "gifts": []
}


STEP 4 — items

items: object

means each element is:

{ }


STEP 5 — object fields

properties:
  name
  price_range
  why_its_good


Final structure becomes:

{
  "gifts": [
    {
      "name": "...",
      "price_range": "...",
      "why_its_good": "..."
    }
  ]
}

------------------------------------------------------------
7. COMMON PROBLEMS
------------------------------------------------------------

Problem → Empty arrays

Cause:
minItems not defined


Problem → {}

Cause:
Used chat.completions instead of responses API


Problem → empty strings ""

Cause:
"type": "string" allows empty text


Problem → model ignores schema

Cause:
- wrong API
- bad prompt
- invalid schema


------------------------------------------------------------
8. HOW LLMs THINK (IMPORTANT)
------------------------------------------------------------

Model objective:

Produce output that satisfies schema
with minimal token cost.

Therefore:

Allowed → []
Model → []

9. GOLDEN RULE FOR STRUCTURED OUTPUT

pattern
minimum
maximum


More constraints → more reliable AI output.
