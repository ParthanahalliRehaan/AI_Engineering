# Gift Genie Frontend Issue: No Output with `marked`

## Problem
1. **Static File Serving**
   - You served files from `__dirname` (the server folder).
   - `index.html`, `style.css`, and `script.js` may not be in that folder.
   - Result: Browser can’t load frontend correctly at `http://localhost:3000/`.

2. **Hidden Output Container**
   - In HTML, the output container starts with `class="hidden"`.
   - You never remove this class in `script.js`.
   - Result: Even if data is fetched, it stays invisible.

3. **Environment Variables**
   - Backend uses `process.env.AI_KEY`, `AI_URL`, `AI_MODEL`.
   - If `.env` is missing or misconfigured, API call fails silently.

4. **`marked` Import/Usage**
   - You wrote `import { marked } from "marked";` in `script.js`.
   - Browsers don’t understand this unless you bundle with Webpack/Vite.
   - Without bundling, the script crashes before updating DOM.
   - Removing `marked` makes everything work → confirms import issue.

## Root Cause
- `marked` is a Node/ESM package designed for environments that support module bundling.
- Browsers cannot directly interpret `import { marked } from "marked";` unless the code is processed by a **bundler** (Webpack, Vite, Parcel, etc.).
- Without bundling, the script crashes before DOM updates, so no output appears.

---

## Bundler Approach
- A **bundler** compiles Node/ESM imports into browser‑friendly JavaScript.
- If you want to use `import { marked } from "marked";`, you must:
  1. Install a bundler (e.g., Vite, Webpack).
  2. Configure it to handle ES modules.
  3. Build your project so the browser receives compiled JS.
- This allows you to keep modern `import` syntax in your code.

---

## CDN Approach
- Instead of bundling, you can load `marked` directly from a CDN.
- Add a `<script>` tag in your HTML pointing to a CDN version (e.g., jsDelivr).
- This exposes `marked` globally in the browser without needing imports.
- Then you can call `marked.parse(...)` in your script.

---

## Simplest Approach
- If your backend already enforces plain text responses, you don’t need Markdown parsing at all.
- Just render the text directly (`textContent` or `innerHTML`) and skip `marked`.

---

## Summary
- **Bundler** → required if you want to use `import { marked } from "marked";`.
- **CDN** → quick fix if you want Markdown parsing without bundling.
- **Drop `marked`** → simplest if you only need plain text output.
