Short Answer

You see:

Cannot GET /api/gift-genie

because:

Your Express server IS running

But the route only supports POST

The browser sends a GET request

So Express responds:

Cannot GET /api/gift-genie

This is normal behavior.



1. What Happens When You Type a URL in the Browser

When you visit:

http://localhost:3000/api/gift-genie

the browser automatically sends:

GET /api/gift-genie

But your server route is:

app.post("/api/gift-genie", async (req, res) => {

Meaning:

POST allowed
GET not allowed

So Express cannot match the route.



2. Express Routing Logic

Express checks routes like this:

Request Method + Path

Example table:

Request                     Route Match
GET  /api/gift-genie       ❌ No match
POST /api/gift-genie       ✅ Match

Because your route is:

app.post("/api/gift-genie")



3. Why Express Shows “Cannot GET”

When no route matches, Express returns a default response:

Cannot GET /api/gift-genie

It means:

Server is running
but this method is not defined

Important:

This is NOT an error.

It is actually proof that the server is working.



4. Why It Works From Your Frontend

Your frontend sends:

fetch("/api/gift-genie", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ prompt: userPrompt })
})

So the request becomes:

POST /api/gift-genie

That matches your Express route, so it works.

6. How You Could Allow Browser Testing

If you wanted the browser to work directly, you could add:

app.get("/api/gift-genie", (req, res) => {
  res.send("API working")
});

Then visiting:

localhost:3000/api/gift-genie

would show:

API working



7. The 20% Knowledge That Explains 80%

Understand this rule:

HTTP method matters.

Routes are defined by:

METHOD + PATH

Example:

GET    /users
POST   /users
PUT    /users
DELETE /users

Each is a different route.
