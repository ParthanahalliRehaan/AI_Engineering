===============================================
PROBLEM 1
===============================================

Marked library failed when using CDN.

Reason:
CDN libraries expose global variables, but the code tried to import it
like an npm module.

Wrong:

import { marked } from "marked"

Solution:
Install locally and use bundler.

npm install marked

Then import normally:

import { marked } from "marked"



===============================================
PROBLEM 2
===============================================

Browser cannot resolve npm packages.

Example error:

Failed to resolve module specifier "marked"

Reason:
Browsers do not understand:

node_modules
npm dependency resolution

Solution:
Use a bundler (Vite) which converts npm dependencies into
browser-readable JavaScript.



===============================================
PROBLEM 3
===============================================

After adding Vite, API requests returned:

POST http://localhost:5173/api/gift-genie 404

Reason:
Frontend runs on port 5173 but backend runs on port 3000.

Browser sent request to:

localhost:5173/api/gift-genie

But the API exists at:

localhost:3000/api/gift-genie



===============================================
SOLUTION
===============================================

Create vite.config.js


import { defineConfig } from "vite"

export default defineConfig({

  root: "client",

  server: {
    proxy: {
      "/api": {
        target: "http://localhost:3000",
        changeOrigin: true
      }
    }
  }

})


This tells Vite:

Forward all requests starting with /api to the Express server.



===============================================
PROBLEM 4
===============================================

Proxy still did not work.

Reason:
Vite was started using:

"dev": "vite client"

When you pass a directory argument, Vite may ignore the root
configuration inside vite.config.js.



===============================================
FINAL FIX
===============================================

Change package.json scripts.

Wrong:

"dev": "vite client"

Correct:

"dev": "vite"

Now Vite reads:

root: "client"

from vite.config.js correctly.



===============================================
FINAL WORKFLOW
===============================================

Step 1

Start backend server

node server/server.js


Express starts on:

http://localhost:3000



Step 2

Start frontend server

npm run dev


Vite starts on:

http://localhost:5173



Step 3

User opens the frontend

http://localhost:5173



Step 4

User submits the form.

script.js runs:

fetch("/api/gift-genie")



Step 5

Browser sends request to Vite:

localhost:5173/api/gift-genie



Step 6

Vite sees the proxy rule:

"/api" → localhost:3000

So it forwards the request.



Step 7

Express receives the request:

POST /api/gift-genie



Step 8

Express calls the OpenAI API and produces a response.

Server returns:

res.json({ reply: ... })



Step 9

The response returns through the same path.

Express → Vite → Browser



Step 10

script.js receives the JSON response.

const data = await res.json()



Step 11

Markdown is converted to HTML.

marked.parse(data.reply)



Step 12

The HTML is injected into the page.

outputContent.innerHTML = marked.parse(data.reply)

===============================================
WHY REMOVING "client" FROM THE SCRIPT WORKED
===============================================

Originally:

npm run dev → vite client

This forces Vite to use the "client" folder as root
from the command line.

Because of that, Vite sometimes ignores or partially
overrides the config file.



After changing to:

npm run dev → vite

Vite reads vite.config.js.

Inside that config we already specified:

root: "client"

So the configuration is now consistent.



===============================================
KEY CONCEPTS (THE IMPORTANT 20%)
===============================================

1. Browsers cannot resolve npm modules.
2. Bundlers convert npm dependencies into browser code.
3. Vite runs a development server for the frontend.
4. Express runs a backend API server.
5. When ports differ, a proxy is required.
6. Vite proxy forwards API requests to Express.
7. The response travels back through the same path.
8. Frontend receives JSON and renders it.
