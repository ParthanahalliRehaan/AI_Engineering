Q7
Think
What is the difference between microtasks and macrotasks in the event loop? Which queue does a resolved Promise go into?
hint ▾
// Architect Hint
Microtasks (Promise callbacks, queueMicrotask) run before the next macrotask (setTimeout, setInterval, I/O). The event loop drains the entire microtask queue after each macrotask. This ordering has real consequences in async code that chains Promises with timeouts.
// Why an Architect Cares
Streaming LLM responses, real-time agent updates, and WebSocket events all interact with the microtask queue. Getting ordering wrong means your UI updates out of sequence.

Think
What is an embedding in plain English? Why does "The movie was great" sit closer in vector space to "The film was excellent" than to "I need an oil change"?
hint ▾
// Architect Hint
An embedding is a list of numbers (vector) that encodes semantic meaning. Texts with similar meaning get similar vectors. The model learns this from co-occurrence patterns in training data — words that appear in similar contexts get similar positions in vector space. Cosine similarity measures the angle between vectors — small angle = high similarity.
// Why an Architect Cares
Understanding embeddings at an intuitive level lets you debug retrieval failures, choose the right embedding model, and explain the system to non-technical stakeholders. It's the foundation of every RAG system.
