AI Eng RAG — Production Grade, Not Tutorial Grade Month 9 · ~40 hrs ▾

Chunking strategies (where most RAG fails)

  • Fixed-size chunking with overlap — baseline, rarely best
  • Recursive character splitting — better for most prose
  • Semantic chunking — chunk by meaning, not character count
  • Document-aware chunking — respect headings, code blocks, tables
  • Parent-child chunking: small chunks retrieved, large context sent to LLM
  • Write an ADR for every chunking decision: "Why 512 tokens, 50 overlap?"

Advanced retrieval

  • HyDE (Hypothetical Document Embeddings) — generate then retrieve
  • Hybrid search: dense (vector) + sparse (BM25) — pgvector + pg_search
  • Reranking: Cohere Rerank or cross-encoder for precision
  • Multi-query retrieval: generate 3 query variants, merge results
  • Metadata filtering: filter by date, source, category before vector search

RAG evaluation — RAGAS framework

  • Faithfulness: does the answer come from retrieved context?
  • Answer Relevance: does it answer the actual question?
  • Context Precision: are retrieved chunks actually relevant?
  • Context Recall: did we retrieve everything we needed?
  • Build an eval dataset of 50+ Q&A pairs — run RAGAS on every iteration

AI Eng Agents — LangGraph, Tool Calling, Multi-Agent Systems Months 10–11 · ~60 hrs ▾

Agent architecture fundamentals

  • ReAct pattern: Reason → Act → Observe loop — implement from scratch first
  • Tool calling: design tools the LLM can use reliably
  • Tool design principles: one tool, one job; explicit input/output schemas
  • Agent memory: in-context vs external (vector) vs episodic
  • Agent planning: task decomposition, subgoal generation
  • Human-in-the-loop: when to interrupt, pause, and resume

LangGraph — stateful agents

  • Nodes, edges, state — the graph mental model
  • Conditional edges: agent decides its own next step
  • Checkpointing: persist agent state across sessions (Cloudflare D1 / Redis)
  • Streaming: stream intermediate steps to UI in real time
  • LangGraph Studio — visual debugging of agent graphs

Multi-agent patterns

  • Orchestrator + subagents: one agent plans, multiple agents execute
  • Supervisor pattern: supervisor routes tasks to specialist agents
  • MCP (Model Context Protocol) — the emerging standard for agent tools
  • Agent evaluation: task completion rate, step count, cost per task

AI Eng Evals, Observability, Fine-tuning & Azure AI Deep Dive Month 12 · ~40 hrs ▾

Evaluation frameworks

  • LLM-as-judge: use a powerful model to score outputs — know its limits
  • RAGAS for RAG pipelines (faithfulness, relevance, precision, recall)
  • Human evaluation: when metrics lie, humans don't
  • Regression test suite: 50–100 inputs that must always pass
  • A/B eval: compare prompt v1 vs v2 systematically, not by gut
  • Evals in CI/CD: fail the deploy if accuracy drops below threshold

Azure AI — go deep, start now

  • Managed identity + RBAC: no API keys in production — Azure AD tokens only

Security OWASP AI Top 10 — Attack Surfaces for LLM Systems Months 13–14 · ~60 hrs ▾

Attack surface understanding

  • LLM06 Excessive Agency: agents acting beyond intended scope — the multi-agent threat
  • LLM08 Vector and Embedding Weaknesses: adversarial inputs that corrupt your pgvector index
  • LLM09 Misinformation: confident hallucinations — how RAGAS evals catch this

Security Secure AI Pipelines — Build the Defences Month 15 · ~30 hrs ▾

Defensive techniques to implement

  • Input validation layer: sanitise before every LLM call, always
  • Output validation: parse and validate LLM JSON with Zod before use
  • Instruction hierarchy: system prompt authority > user input — enforce it
  • Llama Guard: open-source content safety classifier for input/output
  • Azure Prompt Shields: detect injection attempts pre-inference
  • Semantic similarity guardrails: flag inputs resembling known attacks
  • Tool permission scoping: agents get minimum permissions, not maximum
  • HITL gates: irreversible actions require human confirmation
  • Audit logging: every LLM call, tool invocation, and decision node logged

Principles for the 18 Months
Strategic Notes
🔒
Seal the Stack. Mean It.
Frontend is done after Month 3. React + Next.js + Tailwind + Cloudflare. No framework hopping. The discipline of the seal IS the skill. Every senior engineer you want to be has a stack they know deeply, not broadly.
🌊
Each Project Compounds Into the Next
NexusAPI is the gateway OracleRAG calls. ShieldLayer hardens OracleRAG's agents. SentinelAI runs all of it. This isn't five separate projects — it's one system built in layers. Interviewers can trace the entire architecture across four months of decisions.
◉
Azure AI — Go Deep, Not Wide
Most bootcamps teach OpenAI API calls. Almost none teach Azure managed identity, private endpoints, Responsible AI dashboard, or Prompt Shields. Combined with the security phase, you become the engineer enterprises trust. That's the senior hire.
📄
Write an ADR for Every Major Decision
15 minutes, every time: "Why pgvector over Pinecone?" "Why LangGraph over AutoGen?" "Why Redis cache at 0.92 cosine threshold?" The ADR archive on GitHub becomes your portfolio of thinking. Architects are hired for their decisions, not their code.
🧪
Evals First, Not Last
Most engineers write evals after the system breaks. Build your RAGAS suite in Month 9, before you think you need it. "I have 50 eval cases and run them in CI on every deploy" is a sentence that ends interviews in your favour.
🎯
One Project. One Section. Ship the Layer.
Every section, one project evolves. Not a tutorial clone. Not a half-built demo. A real system with a real README and at least one ADR per decision. The habit of shipping layers is the habit of an architect. Everything else is theory.
