# 🚀 Study Guide: Difference Between `npm start`, `npm run dev`, and Others

---

## 1. Step-by-Step Breakdown
Let’s untangle these commands:

1. **`npm start`**
   - Default script in `package.json`.
   - Runs whatever is defined under `"start"`.
   - If no `"start"` script is defined, it runs `node server.js` by default.
   - Commonly used to start production-ready apps.

2. **`npm run dev`**
   - Custom script defined in `package.json` (usually `"dev": "next dev"` or `"dev": "vite"`).
   - Runs the app in **development mode**:
     - Hot reloading
     - Debugging tools
     - Faster builds, but less optimized

3. **Other scripts (`npm run build`, `npm run test`, etc.)**
   - `"build"` → Prepares optimized production files.  
   - `"test"` → Runs test suites.  
   - `"lint"` → Checks code style.  
   - Any custom script can be added under `"scripts"` in `package.json`.

---

## 2. Visual Explanations
Imagine a **flowchart**:

```
npm start → Production server (optimized)
npm run dev → Development server (hot reload, debugging)
npm run build → Creates production bundle
npm run test → Runs tests
```

Or a **mind map**:
- Scripts in `package.json`
  - Start → Production
  - Dev → Development
  - Build → Optimized bundle
  - Test → QA checks

## 3. Analogies & Metaphors
- **`npm start` = Opening the restaurant to customers**  
  Everything is polished, optimized, and ready for real use.  
- **`npm run dev` = Cooking in your kitchen**  
  You experiment, taste, adjust recipes quickly (hot reload).  
- **`npm run build` = Meal prep**  
  You package meals neatly for delivery (optimized files).  
- **`npm run test` = Quality check**  
  Making sure the food is safe and tasty before serving.

## 4. Active Recall Questions
- **MCQ:** What does `npm start` run if no script is defined?  
  A) `node index.js`  
  B) `node server.js`  
  C) `npm run dev`  
  *(Answer: B)*

- **True/False:** `npm run dev` is always a built-in npm command.  
  *(Answer: False – it depends on your `package.json`)*

- **Short Answer:** Which script prepares optimized files for deployment?  
  *(Answer: `npm run build`)*

---

## 6. Summary & Key Takeaways
- `npm start` → Runs production-ready app.  
- `npm run dev` → Runs development mode (hot reload, debugging).  
- Other scripts (`build`, `test`, etc.) → Custom tasks defined in `package.json`.  
- The difference lies in **environment** (development vs production) and **purpose** (testing, building, running).

---

## 7. Application Examples
- **React App:**  
  - `npm run dev` → Starts Vite/Next.js dev server with hot reload.  
  - `npm run build` → Creates optimized static files.  
  - `npm start` → Serves the built files (production server).

- **Node.js Backend:**  
  - `npm run dev` → Uses `nodemon` for auto-restart on code changes.  
  - `npm start` → Runs `node server.js` in production.
